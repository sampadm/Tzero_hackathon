Architecture Design & Implementation Plan
Tzero BYOA
Technical Architecture
Full-stack architecture for the Bring Your Own Asset onboarding platform. Designed for local deployment with live Claude API connections and Ethereum Sepolia testnet — production-grade patterns without cloud infrastructure.
Local-only deployment Live Claude API Sepolia testnet Fully functional flows
Section 01
Architectural Principles
ConstraintImplication
Local-only deploymentDocker Compose replaces cloud infrastructure. No S3, no Lambda, no RDS — equivalents run in containers on localhost.
Fully functional flowsEvery step of the PRD user journey executes end-to-end, including real API calls and real blockchain transactions. No stubs in the happy path.
Live AI (Claude API)API key stored in .env. Calls go to Anthropic's production API. No mocking in the extraction pipeline.
Ethereum testnetAll contracts deploy to Sepolia. Real gas (test ETH), real transaction hashes, real on-chain state.
Design Philosophy
PrincipleRationale
Service separation over monolithFrontend, API, worker, and DB run as independent processes. Mirrors a production deployment and keeps concerns isolated even locally.
Async by defaultAI extraction, contract generation, and blockchain deployment are long-running. They run in a background worker queue — never blocking HTTP responses.
Audit-first data modelEvery state change is a new record, never an in-place update. Immutable audit trail is a PRD compliance requirement.
Security is not optionalSecrets management, input validation, and dependency scanning (Snyk) are in from day one — not retrofitted.
Local → cloud in .env onlyEvery local dependency maps 1:1 to a managed cloud service. Cloud deployment requires no code changes, only environment variables.

Section 06
AI Extraction Pipeline
Step 1 — PDF Text Extraction (pdfplumber)

Layout-aware text extraction handles multi-column NVCA formats. Image-only PDFs are detected and rejected immediately with a user-friendly error — no wasted API calls.

Step 2 — Structured Extraction (Claude API)

Model: claude-sonnet-4-6. Single API call using tool use (structured output) to guarantee parseable JSON. The extraction schema covers 18 fields across four sections.

Pythonai_service.py — extraction schema
class ExtractionField(BaseModel):
    value:        str | None
    confidence:   float     # 0.0 – 1.0
    source_quote: str | None  # verbatim text from document
    source_page:  int | None
    notes:        str | None  # AI reasoning for low confidence

class TermSheetExtraction(BaseModel):
    # Valuation & Economics
    pre_money_valuation:          ExtractionField
    investment_amount:            ExtractionField
    option_pool_pct:              ExtractionField
    dividend_rate:                ExtractionField
    dividend_type:                ExtractionField

    # Liquidation & Conversion
    liquidation_preference_multiple: ExtractionField
    liquidation_participation:    ExtractionField
    conversion_ratio:             ExtractionField

    # Investor Rights
    pro_rata_rights:              ExtractionField
    anti_dilution_type:           ExtractionField
    board_seats:                  ExtractionField

    # Compliance
    investor_type:                ExtractionField
    offering_type:                ExtractionField
    lock_up_period_months:        ExtractionField
Step 3 — Confidence Tiers & Auto-Flag Rules
ConfidenceTierUI TreatmentRequires Review?
≥ 0.85highGreen chip — no action requiredNo
0.60 – 0.84mediumAmber chip — editable but not blockedOnly if required field
< 0.60lowRed chip — edit input shown inline, block on submitYes — must confirm

Additional auto-flag triggers: source_quote is null, or source text contains "see Schedule", "as defined in", or similar cross-references.

Step 4 — Compliance Rule Detection (deterministic)

A rule-based layer runs after extraction — no additional AI call. Evaluates extracted fields against hardcoded rules (Reg D/Reg S, accredited investor, lock-up). Detected rules become direct parameters to the contract template.

Section 07
Smart Contract Architecture
ERC-3643 (T-REX Protocol)

The permissioned token standard for regulated securities. Enforces identity registry (only whitelisted verified investors can hold tokens), pluggable compliance module (transfer restrictions, lock-ups, investor caps), and standard ERC-20 with compliance checks on every transfer.

TemplateSelected When

Deployment Flow (Sepolia)
Flowdeployment_task.py
1. Load bytecode from generated_contracts
2. Connect to Sepolia via Infura/Alchemy RPC (env var)
3. Sign transaction with Tzero deployer wallet (private key in .env)
4. Broadcast → record tx_hash immediately (user can see it)
5. Poll for confirmation (up to 10 blocks, ~2 min)
6. On confirm → record contract_address, block_number, gas_used
7. Update asset.status = 'deployed'

Retry: exponential backoff, max 3 attempts.
After 3 failures → mark deployment.status = 'failed', log to console.

Hardening Summary
ConcernMitigation
XSSReact JSX escaping by default. No raw HTML injection in UI.
CSRFJWT in Authorization header (not cookies) — no CSRF attack surface.
SQL InjectionSQLAlchemy ORM with parameterized queries everywhere — no raw string SQL.
Path Traversalasset_id is UUID — used as filename. No user-controlled path components.
Private Key ExposureNever logged. Never returned in API responses. Loaded only in worker process.
Solidity InjectionTemplate parameters are whitelisted types (int, string, bool). No arbitrary code injection possible.
Audit Log Tamperingaudit_events table: no UPDATE/DELETE permissions granted to app DB user at the PostgreSQL level.
PDF StorageFernet encryption (AES-128-CBC + HMAC-SHA256) at rest. Decrypted only in memory during extraction.
